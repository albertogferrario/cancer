## Milestone Summary

**Key Decisions:**
- JSON fallback opt-in per route via `render_with_json_fallback()` (security consideration)
- `accepts_json()` on InertiaRequest trait for framework-agnostic detection
- Docs URL: docs.ferro-rs.dev (dedicated subdomain for documentation)
- Website URL: ferro-rs.dev (main framework website)

**Issues Resolved:**
- API testing difficulty (now use Accept: application/json header)
- Manual type regeneration needed (now automatic in ferro serve)
- Missing utility types (JsonValue, ValidationErrors now exported)
- Wrong documentation URLs (all fixed to docs.ferro-rs.dev)

**Issues Deferred:**
- Shared Props mechanism (InertiaShared pattern already documented, deferred)

**Technical Debt Incurred:**
None
