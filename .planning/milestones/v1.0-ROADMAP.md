## Overview

Transform the Ferro web framework from developer-centric to agent-first. The journey starts with reducing boilerplate in core patterns (handlers, models, validation), expands MCP introspection for agent comprehension, improves CLI scaffolding for feature-level generation, and concludes with polish that makes the entire framework self-documenting and agent-friendly.

## Milestone Summary

**Key Decisions:**
- Agent-first over developer-first - Non-technical users via agents is the target market
- FerroModel derive on entities - Apply derive to entity files (auto-generated) not model files
- ValidateRules not Validate - Named derive macro `ValidateRules` to avoid conflict with validator crate
- Tool vs Resource for Glossary - Implemented domain_glossary as tool rather than MCP resource
- Structured Description Format - Standardized on When/Returns/Combine format for all tool descriptions
- Category-specific fix prioritization - Fix suggestions have priority 1-5 where 1 is highest

**Issues Resolved:**
- Handler boilerplate reduced with #[handler] macro
- Model boilerplate reduced with FerroModel derive
- Validation syntax streamlined with ValidateRules derive
- MCP expanded to 30+ introspection tools
- CLI scaffolding improved with smart defaults and FK detection
- Actionable error messages implemented

**Issues Deferred:**
- Action injection in handler macro (deferred until macro enhancement)

**Technical Debt Incurred:**
- Pre-existing test failures in ferro-storage (unrelated to v1.0 work)
- Pre-existing metrics test flakiness
