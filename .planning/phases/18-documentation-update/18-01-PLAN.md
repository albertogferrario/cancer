# Plan: User Documentation Rebrand

<objective>
Update all user-facing documentation in docs/src/ to replace "cancer" with "ferro" throughout. This includes CLI commands, import statements, crate names, and framework references.
</objective>

<context>
Phase 17 completed MCP server rebrand. Documentation still references the old "cancer" naming throughout:
- CLI commands: `cancer new`, `cancer serve`, `cancer make:*`, `cancer migrate`
- Imports: `use cancer::*`, `use cancer_events::*`, `use cancer_queue::*`
- Crate names: `cancer-cli`, `cancer-events`, `cancer-queue`
- Framework name in prose: "Cancer framework", "Cancer provides..."

Total: 311 occurrences across 19 files in docs/src/.

Replacement mapping (from Phase 13 audit):
- `cancer` → `ferro` (CLI binary, crate prefix)
- `Cancer` → `Ferro` (framework name in prose)
- `cancer-*` → `ferro-*` (crate names)
- `cancer_*` → `ferro_*` (Rust module imports)
</context>
