# Phase 21 Plan 01: Repository & CI Rebrand

---
phase: 21-repository-ci
plan: 01
wave: 1
depends_on: []
autonomous: true
---

<objective>
Update all GitHub repository infrastructure, CI/CD pipelines, installer scripts, and Cargo.toml repository URLs from cancer to ferro naming. This completes the public-facing infrastructure rebrand before crates.io publication.
</objective>

<tasks>

<task id="1">
<title>Update GitHub Actions release workflow</title>
<files>
- .github/workflows/release.yml
</files>
<action>
Replace all cancer references in release.yml:
1. Build commands: `cancer-cli` → `ferro-cli`
2. Binary paths: `release/cancer` → `release/ferro`
3. Archive names: `cancer-${{ github.ref_name }}` → `ferro-${{ github.ref_name }}`
4. Artifact names: `cancer-${{ matrix.target }}` → `ferro-${{ matrix.target }}`
5. Tar file contents: binary name `cancer` → `ferro`
</action>
<verify>
Read release.yml and confirm no "cancer" references remain
</verify>
<done>release.yml uses ferro naming throughout</done>
</task>

<task id="2">
<title>Update installer scripts</title>
<files>
- scripts/install.sh
- scripts/create-app.sh
</files>
<action>
Update install.sh:
1. REPO variable: `albertogferrario/cancer` → `ferroframework/ferro`
2. BINARY_NAME: `cancer` → `ferro`
3. INSTALL_DIR: `CANCER_INSTALL_DIR` → `FERRO_INSTALL_DIR`, `.cancer/bin` → `.ferro/bin`
4. Comments: "Cancer Framework" → "Ferro Framework"
5. PATH export message: reference ferro instead of cancer

Update create-app.sh:
1. REPO variable: `albertogferrario/cancer` → `ferroframework/ferro`
2. BINARY_NAME: `cancer` → `ferro`
3. Comments: "Cancer Create App" → "Ferro Create App"
4. Usage messages: reference ferro command
</action>
<verify>
Read both scripts and confirm no "cancer" references remain
</verify>
<done>Both scripts use ferro naming and ferroframework org</done>
</task>
